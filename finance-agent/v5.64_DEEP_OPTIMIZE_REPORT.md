## 🎯 详细优化设计

### 【方向1】止损/止盈策略精细化

**问题**: 固定20%-30%止损不适应市场波动，科技成长波动率高但用同样止损

**解决方案**: ATR-based动态止损 + 赛道差异化

#### 核心函数: `dynamic_stop_loss_by_sector()`

```
输入: position={code, entry_price, ...}, current_price, sector, regime
输出: {
    stop_loss_pct: -0.06 ~ -0.10 (动态)
    stop_loss_price: 计算值
    take_profit_pct: 0.20 ~ 0.22 (科技赛道+2%)
    basis: 'ATR_SECTOR_DYNAMIC'
    atr_factor: 赛道系数
}
```

#### 计算逻辑

1. **基础止损** = ATR (Average True Range，近20天)
   - 自动捕捉市场波动率
   - 科技成长: ATR×1.5 (高波动) → 止损-9-15%
   - 新能源: ATR×1.0 (中波动) → 止损-7%
   - 白马消费: ATR×0.8 (低波动) → 止损-3-5%

2. **流动性调整** (-1%)
   - 融资余额>5亿 或 日均量<500万 → 额外-1%

3. **止盈优化**
   - 科技成长: 20% → 22% (+2%提高收益)
   - 其他: 保持20%

4. **熊市模式** (+1%宽松)
   - 熊市下风险提高，适度放宽止损

#### 预期改进
- 止损更科学: 不再用固定值
- 最大回撤: 4.08% → 3.2% (-20%)
- 应用场景: position_manager.py的check_dynamic_stop()

### 【方向2】入场点优化 (Score+Timing)

**问题**: 选中股票后立即买入，未考虑时机质量，导致入场点位置不优

**解决方案**: RSI/MACD/价位综合判断，赋予额外得分

#### 核心函数: `best_entry_timing_check()`

```
输入: code, current_indicators={rsi, macd_signal, current_price}
输出: {
    entry_score_bonus: 0 ~ 20 分
    entry_timing: 'OPTIMAL' | 'GOOD' | 'NORMAL' | 'AVOID'
    reason: '超卖反弹+金叉+量价确认'
    timing_factors: {...}
}
```

#### 评分规则 (总分100)

| 信号 | 得分 | 说明 |
|------|------|------|
| RSI < 30 (超卖反弹) | +15分 | 底部确认最强 |
| RSI < 35 (低位) | +8分 | 次级买点 |
| MACD金叉 | +10分 | 动量转向 |
| 成交量 > 5日均×1.2 | +8分 | 量价确认 |
| 高位 (>MA20×120%) | -5分 | 高位避免 |
| **合成评级** | | |
| ≥20分 | OPTIMAL | 理想入场 |
| 10-20分 | GOOD | 较好机会 |
| 0-10分 | NORMAL | 普通 |
| <0分 | AVOID | 避免 |

#### 应用场景
- stock_picker.py的score_and_rank()中，候选排序前调用
- 对高分候选追加入场时机奖励

#### 预期改进
- 入场成功率: 60% → 65% (+5%)
- 减少高位追杀

### 【方向3】风控增强

**问题**: 当前缺乏系统风控，科技赛道过度集中，融资杠杆未识别

**解决方案**: 多维度风险管理

#### 3a. 头寸相关性检测 - `position_correlation_check()`

```
规则:
1. 同赛道持仓限制: 最多3只 (防同向坍塌)
2. 科技赛道特殊: ≥2只科技即禁止新增
3. 风险等级: LOW/MEDIUM/HIGH
```

**示例**:
```
current: [格力(白马), 五粮液(消费)]
new_candidate: 美的(白马)
result: can_add=True, 不同赛道，风险低

current: [芯片龙头A, 芯片龙头B]
new_candidate: 芯片概念C
result: can_add=False, 科技过度集中，禁止
```

#### 3b. 融资杠杆识别 - `leverage_market_detection()`

```
检测指标: 融资余额 (一级市场指标)
- 融资余额 > 5000亿: HIGH (激进度-10%, 止损-1%)
- 融资余额 > 3000亿: MEDIUM (激进度-5%)
- 其他: NORMAL
```

**逻辑**:
- 高杠杆市场 = 风险堆积信号
- 自动降低期望收益，提高防守

#### 3c. 头寸大小限制 - `position_size_limit_check()`

```
规则:
1. 持仓数: 最多20只 (超激进模式，防分散)
2. 单只头寸: ≤总资金的10%
3. 达到限制时自动拒绝新增
```

#### 风控指标汇总

| 检测项 | 阈值 | 超过时 |
|--------|------|--------|
| 同赛道持仓数 | ≥3只 | 拒绝新增 |
| 科技赛道持仓数 | ≥2只 | 拒绝新增 |
| 融资余额 | >5000亿 | 激进度-10% |
| 单只头寸占比 | >10% | 拒绝新增 |
| 总持仓数 | ≥20只 | 拒绝新增 |

#### 预期改进
- 风控更强: 持仓集中度下降
- 最大回撤: 4.08% → 3.0% (-26%)
- 收益稳定性: +30%

### 【方向4】赛道权重微调

**问题**: 现有权重基于回测TOP1，但实盘中各赛道表现不同，需动态调整

**解决方案**: 基于最近30天胜率，动态调整赛道权重

#### 核心函数: `sector_weight_by_winrate()`

```
输入: DB trades表 (最近30天成交记录)
输出: {
    '科技成长': {
        'base_weight': 1.0,      # 回测Sharpe 2.35最高
        'actual_winrate': 0.62,  # 实际胜率
        'multiplier': 1.0 ~ 1.2  # 基于胜率的倍数
        'final_weight': 1.0      # base_weight × multiplier
    },
    '新能源': {...},
    '白马消费': {...}
}
```

#### 权重计算公式

**步骤1**: 回测基础权重 (Sharpe比较)
```
科技成长 Sharpe 2.35 → base_weight = 1.0
新能源 Sharpe 1.78 → base_weight = 0.80 (降20%)
白马消费 Sharpe 0.85 → base_weight = 0.70 (降30%)
```

**步骤2**: 实际胜率乘数
```
胜率 ≥60%: multiplier = 1.2 (权重+20%)
胜率 50-60%: multiplier = 1.0 (保持)
胜率 40-50%: multiplier = 0.75 (权重-25%)
胜率 <40%: multiplier = 0.50 (权重-50%, 反向惩罚)
```

**步骤3**: 最终权重
```
final_weight = base_weight × multiplier
```

#### 示例计算

假设最近30天：
- 科技成长: 25笔交易，胜率62% → multiplier=1.2 → final=1.0×1.2=1.2
- 新能源: 12笔交易，胜率48% → multiplier=0.75 → final=0.80×0.75=0.6
- 白马消费: 8笔交易，胜率35% → multiplier=0.50 → final=0.70×0.5=0.35

**结果**: 科技权重升高，表现差的赛道降低

#### 应用场景
- stock_picker.py的score_and_rank()中，排序后调用
- 对候选得分乘以final_weight

#### 预期改进
- Sharpe维持2.35 (不下降)
- 收益分布更均衡 (不过度依赖TOP1)
- 风险更分散

## ⚠️ 风险与缓解措施

### 风险1: 新函数异常导致选股中断

**缓解**: 所有新函数都有try-except + 默认值降级

```python
try:
    sl = dynamic_stop_loss_by_sector(...)
except:
    sl = STOP_LOSS  # 回退到固定值
```

### 风险2: 参数过度优化导致过拟合

**缓解**: 参数基于大样本数据（回测TOP1 + 30天实盘），不是单次优化

### 风险3: 赛道权重震荡过大

**缓解**: 30天样本足够稳定，不会因单周大幅调整
