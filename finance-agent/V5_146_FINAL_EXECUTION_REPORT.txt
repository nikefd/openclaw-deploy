【改進1】多層情緒緩存系統

實現: SentimentCache 類 (v5_146_PREMARKET_OPTIMIZE.py)

結構:
  L1 (內存快取)
  ├─ TTL: 30秒
  ├─ 速度: <0.01秒
  └─ 場景: 盤前首次調用後30秒內復用

  L2 (SQLite快取)
  ├─ TTL: 8小時
  ├─ 速度: <0.05秒
  └─ 場景: L1過期/進程間調用

  L3 (默認中性)
  ├─ TTL: 永遠有效
  ├─ 成本: 零
  └─ 場景: 所有快取失敗時

測試結果:
  首次採集: 0.86秒 ✓
  L1讀取:   <0.01秒 ✓ (加速 100 倍)
  L2讀取:   <0.05秒 ✓
  L3讀取:   <0.001秒 ✓

【改進2】動態MACD+RSI參數適應

實現: get_adaptive_macd_rsi_params() 函數

邏輯:
  情緒 <25  (極度恐懼) → 激進模式  (Kelly ×1.30)
  情緒 25-40 (恐懼)     → 樂觀模式  (Kelly ×1.15)
  情緒 40-60 (中性)     → 基準模式  (Kelly ×1.00)
  情緒 60-85 (貪婪)     → 謹慎模式  (Kelly ×0.85)
  情緒 >85  (極度貪婪)  → 高度謹慎  (Kelly ×0.70)

參數變化 (MACD):
  激進: (10,30,7)    [更敏感快速]
  基準: (12,26,9)    [標準設置]
  謹慎: (14,24,11)   [更保守穩定]

參數變化 (RSI):
  激進: period=12, oversold=40, overbought=60
  基準: period=14, oversold=30, overbought=70
  謹慎: period=16, oversold=25, overbought=75

測試結果:
  ✓ 情緒15: 激進模式驗證通過
  ✓ 情緒50: 基準模式驗證通過
  ✓ 情緒95: 高度謹慎驗證通過
  ✓ Kelly係數正確應用
