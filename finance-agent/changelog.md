#### 優化③ 選股異步降級系統 (v5_131_async_fallback.py)
**問題**: stock_picker.py阻塞等待所有數據,任何源超時則整體失敗

**解決方案**:
- 優先級分層: 關鍵(MACD/RSI)>高(成交量/資金)>低(新聞/情緒)
- 分階段超時: 5s(關鍵) → 3s(高) → 1s(低)
- 自動降級: 8秒內確保返回結果

**執行流程**:
```
【第1階段 0-5秒】關鍵數據 (MACD, RSI, 價格)
  ✓ 完成則繼續
  ✗ 超時則標記, 進入第2階段

【第2階段 5-8秒】高優先級數據 (成交量, 資金面)
  ✓ 完成則繼續  
  ✗ 超時則標記, 進入第3階段

【第3階段 8s+】低優先級數據 (新聞, 情緒)
  ✓ 已完成則返回
  ✗ 未完成則跳過, 不影響選股
```

**預期效果**:
- ✅ 盤前啟動: <8秒 (vs不穩定10-30秒)
- ✅ 可靠性: 99% (vs ~80%)
- ✅ 選股完成率: 100% (vs 30-50%失敗)
- ✅ 降級應用率: 成功應對數據源波動
