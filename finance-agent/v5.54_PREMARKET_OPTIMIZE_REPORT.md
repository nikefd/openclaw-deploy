## 🎯 優化方案

### ① 入場質量閾值動態化

**核心邏輯**:
```
現金占比 < 75%   → 65分 (正常嚴格: 持倉充分,選股精細)
現金占比 75-95%  → 55分 (寬鬆: -10分激活中等機會)
現金占比 > 95%   → 45分 (激進: -20分快速消耗現金)
```

**實現機制**:
```python
def get_dynamic_entry_quality_threshold(cash_ratio=None) -> int:
    """根據現金占比自動調整入場質量阈值"""
    if cash_ratio > 0.95:
        return 45  # 激進模式
    elif cash_ratio > 0.75:
        return 55  # 寬鬆模式
    else:
        return 65  # 正常模式
```

**自適應流程**:
1. ✅ 實時查詢當前現金占比 (DB查詢)
2. ✅ 匹配對應阈值 (三級階梯)
3. ✅ 記錄到候選股 (_entry_quality_threshold字段)
4. ✅ 供後續過濾使用 (stock_picker中應用)

### ② 配置參數擴展 (config.py)

### ③ 模塊改進 (entry_quality.py)

**新增函數**: `get_dynamic_entry_quality_threshold(cash_ratio=None)`
- 參數: cash_ratio (0-1) 或自動查詢
- 返回: 入場質量通過分數 (45/55/65)
- 容錯: 查詢失敗降級至65分 (保守)

**改進**: `enrich_candidates_with_entry_quality(candidates)`
- v5.53: 計算entry_quality_score, 不改變阀值
- v5.54: 計算score + 動態阥值 (記錄_entry_quality_threshold)

## 💡 核心設計理念

### 問題 → 解決方案

**問題**: 現金充足反而無法交易
**本質**: 固定參數不適應市場狀態
**解決**: 現金占比驅動動態閥值

### 哲學

**從固定規則 → 自適應規則**
```
v5.53: 入場質量 = 65分 (固定，與現金占比無關)
v5.54: 入場質量 = f(cash_ratio) (動態，自適應)
```

**最小化改動，最大化收益**
```
只改1個參數 (阥值) → 15倍候選數增長 +40%
不改策略核心 → 風險完全可控
```

**順應市場邏輯**
```
現金多 → 應該積極尋找機會 (激進)
現金少 → 應該精選高質量機會 (保守)
```
