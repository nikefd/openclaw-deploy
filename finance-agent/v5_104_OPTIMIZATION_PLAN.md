## 🎯 三層改進方案

### 改進①: 市場情緒信號優化 ⚡
**問題:** get_market_sentiment() 情緒評分因EMA平滑導致滯後  
**現象:** 突發漲停潮(+30只)時，情緒評分仍在65(乐观)，沒有立即跳到80(贪婪)  

**方案:**
1. **新增"急速模式"** - 當10分鐘內新增漲停>10只時，忽略EMA，用原始值
2. **分離"機構資金"信號** - 新增大單買入佔比 (big_buy_count / limit_up_count)  
3. **自適應情緒縮放** - 當情緒極值(>85或<25)時，降低EMA平滑權重

**預期效果:**
- 情緒信號反應時間 5→1秒
- 盤前建倉觸發速度提升50%

---

### 改進②: 動態現金閾值激活 📊
**問題:** 現金激進啟動是定值(95% → 20%)，不考慮市場狀態  
**現象:** 
- 低情緒市場(score<45)時，85%現金激進啟動反而加大虧損風險
- 高情緒市場(score>75)時，還在死守95%才激進，建倉機會喪失

**方案:**  
新增"**情緒聯動現金激活表**":
```
情緒分級    激進啟動% 目標持倉數  安全降速
貪婪(80+)    65%      15只      否
樂觀(65-79)  75%      12只      否
中性(45-64)  85%      8只       是
謹慎(30-44)  92%      5只       是
恐慌(<30)    98%      2只       是(強制)
```

**實施點:** 
- position_manager.py: 新增`get_dynamic_cash_threshold(sentiment_score)`
- stock_picker.py: 調用動態閾值而非固定95%

**預期效果:**
- 高情緒市場建倉速度提升40%
- 低情緒市場風控自動降速，年化虧損減少15%

---

### 改進③: 選股超時防護升級 ⏱️
**問題:** v5.96的45秒→12秒優化缺乏實時監控反饋  
**現象:** 某些高負載情況下仍可能超時，無法自動降級

**方案:**
新增"**智能超時防護層**":
1. **3層候選池動態縮放**:
   - 階段1 (0-5秒): 全量候選(100只)
   - 階段2 (5-10秒): 縮減到50只(按入場質量排序)
   - 階段3 (>10秒): 緊急模式10只(TOP評分)
   
2. **時間預算分配**:
   - 市場情緒採集: 2秒
   - 技術指標計算: 4秒
   - 評分排序: 3秒
   - 緩衝時間: 1秒
   - **總計: 10秒(vs當前可能15-20秒)**

3. **異常降級規則**:
   - 若某步驟超時 → 跳過該步驟用歷史值
   - 若整體>9秒 → 只返回TOP 5候選 (100%確保<10秒)

**預期效果:**
- 超時率 0% → 0%(保持) ✅
- P95完成時間 12秒 → 8秒
- 極端情況完成時間 不確定 → 保證<10秒 ✅
