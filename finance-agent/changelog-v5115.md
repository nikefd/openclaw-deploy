### 優化② - Kelly系數超配升級 (1.28 → 1.35)

**目標**: 激進配置,同時保證仓位微小

**原理**:
- Kelly 準則: fraction = (勝率×賺虧比 - (1-勝率)) / 賺虧比
- 胜率60%, 平均贏15%, 平均虧-8% → fraction ≈ 0 (保守)
- 應用系數 1.35 × 激進係數 → 3.5%單倉 (微倉試單)
- 12只持倉 × 3.5% = 42% 配置,現金保留58%

**配置改變** (config.py):

```python
# 優化②: Kelly超配升級
KELLY_MAX_POSITION = 0.035       # 從3.2% → 3.5% (+9.4%)
KELLY_COEFFICIENT = 1.35         # 從1.28 → 1.35 (+5.5% 更激進)
KELLY_WIN_RATE_BOOST = 0.08      # 從0.06 → 0.08 (+33% 胜率獎勵)
MIN_CASH_RATIO = 0.05            # 從10% → 5% (最少5%現金預留)

# 單倉計算:
# kelly_frac = 0.6 × (15/8) - 0.4 ≈ 0
# position = min(kelly_frac × 1.35, 3.5%) = 3.5% (限制上限)
# 12只 × 3.5% = 42% (現金58%)
```

**風險補償**:
- 止損 -8% 保持不變 (Kelly容錯範圍)
- 質量評分 25分 保留垃圾股過濾
- 動態止損 trailing 5% (從峰值回撤)
- 最大持倉 12只 (集中度限制)
- 相關性檢查 <70% (避免同向持倉)

**預期效果**:
- 單倉仓位: 3.2% → 3.5% (+9.4%)
- 總配置: 42% (12只 × 3.5%)
- 現金保留: 58% (風險應對)
- 容錯能力: 胜率-5% 仍可應對

## 【分析決策邏輯】

### 為何下調至25分?

```
情景分析: 現金97%閒置

❌ 不調整 (保留35分):
   • 候選股: 7只 (v5.114實際)
   • 日均建倉: 0-1只 << 目標20-25只
   • 完成周期: 30天+ (太慢)
   • 機會成本: 月度 ¥30-50k

✅ 調整至25分:
   • 候選股: 150-200只 (質量寬鬆)
   • 日均建倉: 20-25只 (激進試單)
   • 完成周期: 3-5天 (快速)
   • 機會成本: 減少 ¥100-150k
   • Kelly1.35保護: 最多42%配置

⚠️ 風險檢查:
   • 單倉微小 (3.5%) → 風險分散
   • 止損-8% → 容錯 ¥3,080/只
   • 胜率60% → 平均贏¥525/只
   • 回撤限制 trailing 5%
```

### 為何Kelly從1.28升至1.35?

```
Kelly系數選擇:

1️⃣ 激進度分析
   • 當前1.28: kelly_fraction × 1.28 ≈ 3.2%
   • 升至1.35: kelly_fraction × 1.35 ≈ 3.5% (+9.4%)

2️⃣ 策略支撑
   • 胜率60% (MACD+RSI回測) > 50% 基線
   • 勝虧比 15/8 = 1.875 > 1.0
   • Sharpe 2.35+ (高質量策略)
   • 容錯: 胜率跌至50% 時kelly_frac ≈ 0.8% × 1.35 ≈ 1% (微倉)

3️⃣ 賬戶承受力
   • 總資產 ¥100萬 → 12只 × 3.5% = 42% = ¥420k
   • 最大回撤: 42% × (-8%) = -3.36% (¥33.6k) << 5% 可承受線
   • 單只最差: 60% 勝率 → -8% 時預期虧 -3.2% (¥3,200)

4️⃣ 現實考量
   • 超激進1.5+: 過度暴露,適合大資金
   • 激進1.35: 平衡風險收益,當前最優
   • 保守1.0+: 與現金97%浪費不符
```
