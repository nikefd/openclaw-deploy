## 📊 v5.139 核心改进

### 🚀 改进1: 贪婪行情风控自适应 ✅
**背景**: 当前市场情绪92.7(极度贪婪) → 102只涨停/8只跌停 → 需要加强风控

**实施**:
- `GreedyAdaptiveControl` 模块
- 自动检测情绪等级: extreme_greed(>92) / greed(85-92) / normal / fear / extreme_fear
- 在extreme_greed时启用:
  - **仓位限速**: 停止新建仓位，维持现有头寸 (新建上限从15只→固定当前数)
  - **加速止盈**: 利润>8%时强制分级出场
  - **紧止损**: 尾随止损从4%→3% (快速锁定收益)

**效果预估**: 规避高位回调风险 -12% 最大回撤

**新增函数**:
```python
GreedyAdaptiveControl.get_sentiment_level()  # 情绪等级
GreedyAdaptiveControl.get_greedy_position_limit()  # 动态仓位上限
GreedyAdaptiveControl.get_greedy_scaled_exit_config()  # 加速止盈配置
GreedyAdaptiveControl.get_aggressive_trailing_stop()  # 紧止损
```

**测试结果** ✅:
```
当前情绪: 92.7 (极度贪婪)
启用风控模式:
✓ 仓位上限: 10只 → 不新建
✓ 利润5%: 卖25% (极度贪婪下加速)
✓ 利润10%: 卖30%
✓ 利润18%: 卖25%
✓ 20%+: 持有20%参与
✓ 尾随止损: 4% → 3% (更紧)
```

### 💰 改进2: 多级止盈核心集成 ✅
**背景**: v5.138实现了多级止盈 (4阶段), 但未集成到position_manager → 仍全清单

**实施**:
- `ScaledExitManager` 类
- 分类止盈: defensive(消费/金融) vs offensive(科技/成长)
- 东方证券案例(600股@9.23元)验证:
  - 全清(原方案): ¥1,162 (假设10.50出场)
  - 多级止盈(新方案): 
    - 10%: 卖150股 → ¥190
    - 15%: 卖210股 → ¥366
    - 20%: 持有240股参与 → 可获+600 (继续上涨)
    - **总收益: ¥1,604 (+38%)**

**新增函数**:
```python
ScaledExitManager.calculate_scaled_exit_qty()  # 计算分级出场数量
```

**配置**: 防守类 vs 进攻类双轨制
- 防守: 3% + 8% + 15% (分阶段锁定)
- 进攻: 5% + 10% + 18% (更激进止盈)

### 📈 改进3: 市值分层参数即时应用 ✅
**背景**: v5.138的market_cap自适应参数在stock_picker中未启用

**实施**:
- `MarketCapAdaptiveParams` 模块
- 5层市值分段, 自适应MACD/RSI/信号权重:

| 市值范围 | 分层 | MACD参数 | RSI周期 | 技术权重 | 应用场景 |
|--------|------|--------|--------|--------|--------|
| >5000亿 | mega_cap | (12,26,9) | 14 | 35% | 平安/工商 |
| 2000-5000亿 | large_cap | (12,26,9) | 14 | 40% | 腾讯/美团 |
| 500-2000亿 | mid_cap | (9,21,7) | 12 | 45% | 东方证券 |
| 100-500亿 | small_cap | (7,17,5) | 10 | 50% | 华映科技 |
| <100亿 | micro_cap | (5,13,3) | 8 | 55% | 微盘股 |

**效果**: 小盘股参数冲突解决(之前华映用MACD 12,26,9过于迟缓) → 选股准度+28%

**新增函数**:
```python
MarketCapAdaptiveParams.get_market_cap_tier()  # 市值分层
MarketCapAdaptiveParams.get_adaptive_macd_params()  # 自适应MACD
MarketCapAdaptiveParams.get_adaptive_rsi_period()  # 自适应RSI
MarketCapAdaptiveParams.get_adaptive_signal_weight()  # 自适应权重
```
