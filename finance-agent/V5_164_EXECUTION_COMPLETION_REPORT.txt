### 2. 解决方案 (三维度)

#### 维度A: 混合策略融合
```python
# 原方案
entry_score = MACD_RSI_SCORE * 2.35  # 单策略

# 新方案
entry_score = (
    MACD_RSI_SCORE * 0.50 +      # 50% 主策略
    MULTI_FACTOR_SCORE * 0.30 +  # 30% 次策略  
    MA_CROSS_SCORE * 0.20        # 20% 辅助策略
)

# 融资异变时自适应

    weights = {MACD_RSI: 0.30, MULTI_FACTOR: 0.50, MA_CROSS: 0.20}
```

**期望效果**:
- ✅ 无单点故障 (多策略互补)
- ✅ 信号生成 +40%
- ✅ 胜率稳定 58-62%

#### 维度B: 融资缓存优化  
```python
# 原方案
margin_data = ak.stock_margin_data(symbol)  # 延迟 >6小时 ❌

# 新方案 (三层降级)
Layer 1: 快速返回缓存 (<10ms)
Layer 2: 返回过期缓存 (保证有数据)
Layer 3: 返回历史中位数 (最后兜底)
```

**期望效果**:
- ✅ 响应时间 <300ms (vs 6小时)
- ✅ 融资异变激活率 +80%
- ✅ 建仓频率 +35%

#### 维度C: 动态入场门槛
```python
# 原方案
if idle_mode_triggered:
    threshold = 4  # 固定

# 新方案
base = {
    win_rate > 65%: 4,    # 激进
    55% < win_rate < 65%: 5,  # 均衡
    win_rate < 55%: 6     # 保守
}

margin_adj = -2 if margin_anomaly_score > 0.7 else 0
emotion_adj = {
    sentiment < 30: -3,   # 极度恐惧→激进
    sentiment > 92: +2    # 极度贪婪→保守
}

final_threshold = max(3, min(8, base + margin_adj + emotion_adj))
```

**期望效果**:
- ✅ 自适应市场状态 [3-8分]
- ✅ 高胜率期激进 (4分) | 低胜率期保守 (6-8分)
- ✅ 建仓频率稳定 2-3次/周
