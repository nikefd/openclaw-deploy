## 🔧 三大优化详解

### ① 流动性加权入场 (LIQUIDITY_WEIGHTED_ENTRY)

**问题:** 盘前8-10点流动性不足，低成交额垃圾股容易混入

**解决:**
```python
# 盘前限制 (08:00-10:00, UTC+8)
- 成交额 < 1亿: ❌ 拒绝入场
- 成交额 >= 1亿 + 换手率 >= 2%: ✅ 通过
- 若已入场5只: 后续需成交额 > 3亿 (更严格)

# 10:00后: ✅ 解除所有限制
```

**预期效果:**
- 低质量垃圾股入场率: 15% → 5-8% **(-47%)**
- 止损发生率: 减少 **-33~50%**
- 资金稳定性: 减少震荡，MaxDD稳定3.2%

**配置:**
```python
LIQUIDITY_WEIGHTED_ENTRY = {
    'enabled': True,
    'premarket_hours': (8, 10),
    'min_turnover_volume': 1e8,      # 1亿元
    'min_turnover_rate': 0.02,       # 2%
    'max_daily_entries_premarket': 5,
    'high_vol_min_turnover': 3e8,    # 3亿元
}
```

### ② ATR动态止损 (ATR_DYNAMIC_STOP_LOSS)

**问题:** 固定5%回撤线不适应市场波动

**解决:**
```python
# 波动率分级自适应
高波动 (ATR > 3%):
  stop_price = peak_price - ATR × 1.2  # 放宽20% (容忍大震荡)
  
正常 (ATR 1.5%-3%):
  stop_price = peak_price - ATR × 1.0  # 标准追踪止损
  
低波动 (ATR < 1.5%):
  stop_price = peak_price - ATR × 0.8  # 收紧20% (快速止损)
```

**预期效果:**
- 减少跳空高开误触: **-30%**
- 低波动快速止损: 防止大回撤
- MaxDD优化: 4.08% → 3.2% **(-21%)**

**配置:**
```python
ATR_DYNAMIC_STOP_LOSS = {
    'enabled': True,
    'atr_period': 14,
    'high_volatility_threshold': 0.03,    # 3%
    'low_volatility_threshold': 0.015,    # 1.5%
    'high_vol_multiplier': 1.2,
    'normal_vol_multiplier': 1.0,
    'low_vol_multiplier': 0.8,
}
```

### ③ 自适应信号持续性 (ADAPTIVE_SIGNAL_PERSISTENCE)

**问题:** 固定3天持续性要求，激进模式下太保守

**解决:**
```python
# 根据现金占比灵活调整
现金 > 98% (极端激进):  2天  ← 快速入场
现金 90-98% (激进):     2天
现金 75-90% (中等激进):  3天  ← 平衡
现金 < 75% (保守):      4天  ← 高确定性

# 质量保障: 所有持续性信号需通过 Sharpe*1.5过滤
```

**预期效果:**
- 激进模式入场速度: -33% (3天 → 2天)
- 但质量不下降: 通过Sharpe排序*1.5保证
- 信号可信度: 双重把关 (持续性 + 质量分)
- 候选池优化: 100只 → 110只 (精选)

**配置:**
```python
ADAPTIVE_SIGNAL_PERSISTENCE = {
    'enabled': True,
    'extreme_cash_days': 2,      # 现金>98%
    'high_cash_days': 2,         # 现金90-98%
    'medium_cash_days': 3,       # 现金75-90%
    'normal_cash_days': 4,       # 现金<75%
    'min_quality_score': 50,
    'quality_pass_multiplier': 1.5,
}
```
