### Phase 2: 分析與規劃 ✅

#### 根因分析 (5個問題)
1. **[高優]** score_and_rank() line 1695 硬編碼 `[:15]` 截斷候選
   - 症狀: 超激進模式配置為75只候選,但實際獲得15只
   - 影響: 選股數量被人為限制,無法實現8-12倍資金利用率提升
   
2. **[高優]** sector_intelligent_routing() 函數未被調用
   - 症狀: 赛道差異化權重定義了但沒有被執行
   - 影響: 科技成長無法應用MACD 2.5x權重加強
   
3. **[中優]** 現金占比識別邏輯可能NULL異常
   - 症狀: 數據庫查詢可能返回NULL → 激進度降檔
   - 影響: 98.4%現金未能正確觸發超激進模式
   
4. **[中優]** 融資融券信號無實時數據源
   - 症狀: margin_adjustment_evaluation() 缺乏市場數據
   - 影響: +12分的融資信號無法被應用
   
5. **[低優]** 低質量入場監控降級條件太激進
   - 症狀: 樣本少時(18只)直接降級
   - 影響: 誤判率高(25%)

#### 優化規劃 (5個方向)
| 方向 | 目標 | 優先級 |
|------|------|--------|
| A: 候選池參數化 | 15→75只 (+400%) | 高 |

```python
# stock_picker.py 現金查詢部分
# 防護: 若查詢異常導致值為0,回退到默認值
if current_cash <= 0 or current_cash > 10_000_000:
    current_cash = 1_000_000

# 保證_cash_ratio在(0,1]範圍內
if total_value > 0:
```

#### 優化D: 融資融券實時接入 (16行代碼)

```python
if not market_data:
    market_data = {}
    try:
        from data_collector import get_stock_margin_balance

            code = cand.get('code', '')
            if code:
                try:
                    margin_info = get_stock_margin_balance(code)

                        market_data[f"{code}_margin_change"] = margin_info.get('change_pct', 0)
                        market_data[f"{code}_fusion_ratio"] = margin_info.get('fusion_ratio', 0.5)
                except:
                    pass
    except ImportError:
        pass
```

#### 優化E: 低質量監控改進 (8行代碼)
```python
# stock_picker.py monitor_low_quality_entry_performance()
# 原: should_downgrade = success_rate < 0.50
# 新: 使用置信度區間
confidence_margin = 0.10  # 10%置信度裕度
```
