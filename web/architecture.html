1. 总体概览

本系统是一个以 OpenClaw 为核心 AI Agent 框架，搭配自建前端和多个独立 Agent 模块的个人 AI 助手平台。部署在 Azure VPS 上，通过 zhangyangbin.com 提供服务。

核心组成

  • OpenClaw Gateway — AI Agent 运行时 & 多通道消息网关
  • 自建 ChatGPT 风格前端 — 替代 OpenClaw 原版 dashboard
  • 独立 Agent 服务 — 金融、攀岩、AI资讯等专业 Agent
  • Nginx 反向代理 — 统一入口 + SSL + 认证
  • PostgreSQL — 数据持久化

2. OpenClaw 框架

OpenClaw 是一个多通道 AI 网关（Multi-channel AI Gateway），不只是 Agent 框架，更是一个完整的 AI 基础设施：

核心能力

  • Agent 运行时 — 管理 AI Agent 的生命周期、会话、记忆（MEMORY.md）、人格（SOUL.md）
  • 多通道接入 — 支持 Discord、Telegram、Signal、WhatsApp、Web 等渠道接入同一个 Agent
  • 工具系统（Skills） — 可扩展的技能插件，Agent 通过 SKILL.md 获取工具使用说明
  • Cron 调度 — 内置定时任务，支持定时提醒、周期巡检等
  • 心跳机制 — 通过 HEARTBEAT.md 实现主动巡检
  • 子 Agent 编排 — 可 spawn 隔离子会话，支持 ACP 协议接入 Codex 等外部 Agent
  • OpenAI 兼容 API — 对外暴露 /v1/ 接口，兼容 OpenAI API 格式

5. 前端（自建聊天界面）

⚠️ 重要：我们使用的是自建 ChatGPT 风格前端，不是 OpenClaw 原版 dashboard。原版 dashboard 通过 /dashboard/ 路径仍可访问，但主界面完全自建。

页面清单

技术栈

  • 纯 HTML/CSS/JS，无构建工具，单文件架构
  • SSE (Server-Sent Events) 实时流式响应

6. 独立 Agent 模块

🐶 狗蛋（主 Agent）

OpenClaw 的核心 Agent，通过 Gateway 运行。支持多通道（Web、Discord 等）。

  • 人格：随和、直接、偶尔贱兮兮
  • 记忆系统：MEMORY.md（长期）+ memory/*.md（每日）
  • @mention 路由：通过 @攀岩教练、@金融Agent 切换角色

🧗 攀岩教练 Agent

  • 触发方式：@攀岩教练 mention
  • 后端 API：Agents API (:7685)
  • 数据源：~/agent-data/fitness/sessions.json
  • 功能：训练记录、瓶颈分析、训练计划制定
  • 面板：/agents/ → 攀岩教练 tab

💰 金融 Agent（A股）

  • 触发方式：@金融Agent mention / /finance 页面
  • 后端 API：Finance API (:7684)
  • 功能：A股分析、回测、策略推荐
  • 面板：/finance
  • 状态：🔥 最高优先级开发中

📰 AI 前沿追踪 Agent

  • 数据源：~/agent-data/ai-news/
  • 功能：AI 领域资讯聚合、分 track 追踪
  • 面板：/agents/ → AI 前沿追踪 tab

💼 面试准备 Agent

  • 数据源：~/agent-data/interview/
  • 功能：面试题库、SQL 练习、知识点整理
  • 面板：/agents/ → 面试准备 tab

🏘 Agent Village（演示项目）

  • 路径：/demos/village/
  • 后端：Village API (:3100)
  • 功能：多 Agent 协作演示
  • 状态：✅ 已完成
  • 认证：独立的 demo access code 认证

9. 认证系统

主站认证

  • Auth Server (:7683) 负责登录验证
  • 登录后设置 Cookie
  • Nginx 通过 auth_request 对每个请求做 Cookie 校验
  • 未认证自动跳转 /login

Demo 认证（独立）

  • 独立的 demo access code 机制
  • 通过 /_internal_demo_auth 校验
  • 仅用于 /demos/ 路径

API 认证

  • /v1/ 接口支持 Bearer Token 认证
  • /v1/responses 特殊处理：Cookie 认证 + 自动注入 Token

10. 部署与运维

环境

  • 服务器：Azure VPS (Ubuntu, Linux 6.17.0)
  • Node.js：v22.22.0
  • SSL：Let's Encrypt (certbot 自动续期)

服务管理

# OpenClaw Gateway
systemctl --user status openclaw-gateway
openclaw gateway restart

# 系统级服务
sudo systemctl status finance-api
sudo systemctl status agent-village-api
sudo systemctl status nginx

# 手动管理的服务 (file-api, agents-api, usage-api)
# 通过 nohup 启动，需手动重启

前端部署流程

# 1. 修改前端文件
vim /var/www/chat/index.html

# 2. 同步到 deploy repo
cp /var/www/chat/*.html ~/openclaw-deploy/web/

# 3. 提交推送
cd ~/openclaw-deploy && git add -A && git commit -m "update" && git push

OpenClaw 更新

# 更新 OpenClaw
npm update -g openclaw

# ⚠️ npm update 会覆盖 dist，需重新应用 patches
openclaw gateway restart
